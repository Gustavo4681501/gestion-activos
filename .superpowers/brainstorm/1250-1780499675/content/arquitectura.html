Sección 1 — Nueva arquitectura de componentes

Así quedará organizado el código. Cada caja es un archivo con una sola responsabilidad.

Estructura de archivos — src/
📁 pages/
📄 home/Home.jsx
Solo orquesta componentes.
Ya no tiene lógica ni estados del form.
📄 activoDetalle/ActivoDetallePage.jsx
Misma página, más limpia.
Botón "← Volver" añadido.
📁 hooks/
🪝 useActivos.js
Toda la lógica de datos:
cargar, crear, eliminar, filtrar.
Los componentes solo llaman funciones.
📁 services/
⚙️ activos.service.js
Sin cambios grandes.
Se añade manejo de errores.
📁 components/
📊 StatsPanel.jsx
4 tarjetas: Total, Asignados,
Bodega, Dañados/Reparación.
🔍 SearchBar.jsx
Input de búsqueda + selects
de Estado y Categoría.
🗃️ AssetGrid.jsx
Grid de cards. Recibe array filtrado.
Muestra estado vacío si no hay resultados.
🃏 AssetCard.jsx
Una card de activo.
Click → navega al detalle.
📝 AssetForm.jsx
Formulario de registro completo
extraído de Home.jsx.
🖊️ ModalFirma.jsx
Sin cambios funcionales.
Solo mejora visual menor.
🔔 Toast.jsx
Reemplaza alert() y confirm().
Notificaciones no bloqueantes.
Flujo de datos: Home.jsx → useActivos (lógica) → componentes (solo renderan). El hook expone: activos, activosFiltrados, loading, crearActivo(), eliminarActivo(), setFiltros().
